💻 In 2018, I didn’t even know how to properly shut down a computer.
🚀 Today, I’ve started documenting my journey to becoming a DevOps Engineer.

Like many people in tech, my journey started with curiosity and a lot of learning from scratch.

I began by understanding the basics:
🐧 Linux fundamentals
💻 Programming concepts
📝 Coding editors & development tools
🔧 Version control with Git
Step by step, things started making more sense.

Later, I moved into HCLSoftware, where I worked in the MyXalytics R&D development team.

Working in a product-based environment exposed me to how real software products are built, maintained, and continuously improved.
During this time, I also worked around production systems where I saw the importance of:

⚙️ Deployment automation
📊 Monitoring and observability
🛡️ System reliability and stability

That experience changed how I started looking at technology.
I realized that behind every stable product running in production, there is a lot happening — automation, monitoring, infrastructure, and collaboration between teams.
That curiosity kept growing.
And eventually, it led to a decision.

Instead of learning silently, I decided to document my DevOps journey publicly — sharing experiments, mistakes, projects, and lessons along the way.

Through this journey I’ll be exploring:

🐧 Linux for DevOps
🐳 Docker containerization
☸️ Kubernetes clusters
⚙️ CI/CD pipelines
☁️ AWS infrastructure
🧪 Hands-on DevOps projects and troubleshooting

And today, I published my first blog sharing how this journey started — from knowing nothing about computers to beginning a DevOps journey.

🔗 Read it here:

https://rahuldevopsjourney.hashnode.dev/from-not-knowing-how-to-shut-down-a-computer-to-starting-my-devops-journey
